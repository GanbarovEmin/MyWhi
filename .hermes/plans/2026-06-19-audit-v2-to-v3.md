# MyWhi Audit — v2.0 → v3.0 Path

> **Дата:** 2026-06-19
> **Аудитор:** Hermes
> **Объём:** ~6 833 строк Swift, 44 файла, 65 tests pass
> **Цель:** определить, как довести MyWhi до уровня Wispr Flow по UX/плавности/дизайну

---

## TL;DR

MyWhi v2.0 — это уже **production-quality** база: своя дизайн-система (Cohere-inspired), две сцены (menu bar + desktop), AVAudioEngine с pre-roll, WhisperKit-движок, Markdown-vault. Wispr Flow-уровня не хватает **трёх вещей**:

1. **Live streaming transcription** — текст появляется пока говоришь, а не после стопа
2. **Тактильный feedback везде** — haptics, плавные переходы, sound feedback (chime при стопе)
3. **Полировка деталей** — Tailwind-style density, focus rings, micro-animations, dark mode, on-screen keyboard hints

Эти 3 направления разворачиваются в **20 конкретных предложений** ниже.

## 3. Performance bottlenecks

### 3.1 Build/render performance (SwiftUI)

**Найдено:**

1. **`MainPopoverView.swift:21-23`** — `private var recentNotes: [TranscriptNote] { Array(statsObserver.notes.prefix(3)) }` — вычисляется на каждом ререндере popover. При `notes` большом (1000+) это O(n). Должно быть `@State` с кешем или `lazy var` или просто инлайн `ForEach(statsObserver.notes.prefix(3), …)`.

2. **`SettingsViewDesktop.swift:357`** — `await MainActor.run { self.vaultSize = bytes }` после `await appState.vaultStore.sizeOnDisk()`. Лишний hop, `sizeOnDisk()` уже async, можно прямо `self.vaultSize = bytes`.

3. **`AppState.swift:67-72`** — `recorder.$currentLevel.receive(on: RunLoop.main).sink` — Combine pipeline, лучше `@Observable` (если macOS 14+) или `ObservableObject` с прямым `@Published`.

4. **`HDWaveformView.swift:65`** — `TimelineView(.animation(minimumInterval: 1.0 / 30.0))` + `Canvas { ctx, size in … }` — **30Hz опрос через TimelineView** + ре-калькуляция sin-волны 12-32 bars каждый кадр. На retina (60-120Hz) это 60-120 Canvas redraws/sec, при этом actual audio data меняется только 30Hz. **Wasteful**, плюс sin-wave без реального samples.

5. **`MainPopoverView.swift:354-368`** — `RecordingDurationView` использует `TimelineView(.periodic(from: startTime, by: 0.5))` — **тикает каждые 0.5 сек**, перерисовывает Text. Можно сразу `Text(startTime, style: .timer)` — нативный API без overhead.

6. **`InsightsView.swift:107-122 StreakHeatmapView`** — строится на каждом ререндере, `.frame(height: 140)` + HorizontalScrollView + 26 недель × 7 дней = 182 cells. Ререндер дорогой на 100+ нотах. Нужен `@State private var gridCache: [[Int]]?`.

7. **`HomeView.swift:80-86`** — `wordsToday()` фильтрует `statsObserver.notes` на каждом ререндере. O(n) per frame, должно быть precomputed.

8. **`DesktopRootView.swift:51-67`** — `NavigationSplitView` + `List(selection:)` с `HDSidebarItem` — sidebar полностью пересоздаётся при смене selection (нет `LazyVStack`).

### 3.2 Audio performance

**Хорошее:**
- ✅ Engine warm — pre-roll запущен с момента запуска app
- ✅ Level timer 30Hz, не 60-120Hz
- ✅ File write через serial queue (no contention)

**Найдено:**

9. **`AudioRecorder.swift:309`** — `let newSamples = Array(UnsafeBufferPointer(start: channelData, count: frameCount))` — **копирует** все samples в heap array на каждом tap (каждые ~85ms). Для pre-roll ОК, для file write — можно было бы использовать `UnsafeBufferPointer` напрямую в converter. Минор.

10. **`AudioRecorder.swift:350-409 writeSamplesToFileOnQueue`** — создаёт `AVAudioPCMBuffer` для каждого tap, копирует samples, ресэмплит, пишет. **Можно использовать AVAudioFile's input format напрямую** + использовать `AVAudioConverter.inputBuffer` reused pool — снизит GC pressure на длинных записях (>5 мин).

11. **`AudioRecorder.swift:413-428 startLevelTimer`** — `Timer(timeInterval: 0.033, repeats: true)` + `Task { @MainActor in … }` — создаёт новый Task каждые 33ms. На 30Hz это **30 Task allocations/sec**. Можно перейти на `CADisplayLink` или просто polling в `MainActor.run` лупе.

### 3.3 Transcribe latency

12. **`WhisperKitTranscriber.swift:140-143`** — `await pipe.transcribe(audioPaths: [audioPath], decodeOptions: options)` — single batch call, не streaming. Это **главный источник** latency: 5-сек аудио = 5-10 сек processing.

13. **`AppState.swift:264-268`** — между `engineManager.transcribe(...)` и записью в vault нет streaming callback. Один большой `await`.

## 5. UX gaps vs Wispr Flow

### 5.1 Floating HUD position
**Текущее:** top-of-screen, centered, 380×86 panel.
**Wispr Flow:** bottom-of-screen, anchored to active window bottom, slightly translucent.
**Gap:** Top HUD конкурирует с menu bar; bottom ближе к user attention (где текст печатается).
**Suggestion:** оставить top как default, добавить настройку в Settings (top/bottom).

### 5.2 Live streaming — текст появляется пока говоришь
**Текущее:** stop → transcribe → copy.
**Wispr Flow:** partial text обновляется каждые 0.3-0.5 сек.
**Gap:** **HIGH** — основной UX-понт диктовки. Пользователь видит, что его слова поняли.
**Тех. план:** WhisperKit поддерживает streaming через `audioEncoder` callback — реализация в `WhisperKitTranscriber.swift` плюс `LiveTranscriptionView` в HUD.

### 5.3 Inline editor перед paste
**Текущее:** transcript сразу в clipboard.
**Wispr Flow:** transcript появляется в HUD, user может редактировать, потом нажимает insert.
**Gap:** MEDIUM. Power-user фича.
**Тех. план:** добавить `editorMode` в `MainPopoverView` — `TextEditor` вместо auto-copy, отдельная кнопка "Insert".

### 5.4 Sound feedback
**Текущее:** только haptic (на Mac без ForceTouch trackpad — no-op).
**Wispr Flow:** soft chime при старт/стоп.
**Gap:** LOW, но cheap improvement.
**Тех. план:** `AVAudioPlayer` + 2 .aiff файла в Resources (~10KB каждый).

### 5.5 Push-to-talk vs toggle
**Текущее:** `Cmd+Option+D` toggle.
**Wispr Flow:** fn×2 = push-to-talk (hold to record).
**Gap:** MEDIUM. Toggle иногда ловит юзера в записи.
**Тех. план:** добавить опциональный push-to-talk mode (key-down start, key-up stop) через NSEvent monitor.

### 5.6 Real dark mode
**Текущее:** tokens определены, но не читаются из environment.
**Wispr Flow:** полный dark mode.
**Gap:** MEDIUM.
**Тех. план:** сделать `HDColor` полностью theme-aware (см. предложение 1).

### 5.7 Полноценный поиск в Scratchpad
**Текущее:** substring search через `IndexStore.search`.
**Wispr Flow:** fuzzy + filters (date range, language, "today only").
**Gap:** LOW.
**Тех. план:** добавить фильтры по дате (today/week/month/all) над search field.

### 5.8 Voice commands ("new line", "period", "stop recording")
**Текущее:** нет.
**Wispr Flow:** есть, через Whisper prompt.
**Gap:** LOW.
**Тех. план:** `promptPrefix` в `DecodingOptions` + custom vocabulary.

### 5.9 Per-app tone
**Текущее:** нет.
**Wispr Flow:** Slack = casual, Email = formal.
**Gap:** SKIP (over-engineering для v3).
